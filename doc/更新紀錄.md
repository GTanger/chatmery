# Chatmery 更新紀錄

本文記錄各版次功能與行為變更，便於對照與調教。

---

## 1.3.4（對話與記憶：對照 Open WebUI / AnythingLLM 建議）

- **近期輪次可配置**：`CHATMERY_RECENT_TURNS`（1–100，預設 20），送進模型的對話輪次上限可由 tuning 或環境變數設定。
- **解構納入附檔／網搜摘要**：每輪對話儲存時一併存該輪的附檔摘要與網搜摘要（各最多 200 字）；每 20 輪濃縮成 1 句時，prompt 含「[附檔摘要]」「[網搜摘要]」，讓濃縮句可涵蓋依據來源。
- **本輪注入記憶回傳**：SSE 結束前多送一筆 `memory_used`（陣列：layer、preview 各 80 字內），前端可選顯示「本輪使用了哪些記憶條目」。
- **可關閉自動寫入短期／長期**：`CHATMERY_MEMORY_AUTO_SHORT=false` 不執行每 20 輪濃縮進短期；`CHATMERY_MEMORY_AUTO_LONG=false` 短期滿時不觸發長期→核心→靈魂鏈，僅寫入短期。
- **文件**：記憶階層／記憶模式 doc 改為「最近 N 輪（CHATMERY_RECENT_TURNS，預設 20）」；[對話與記憶—對照 Open WebUI、AnythingLLM 的建議](對話與記憶—對照%20Open%20WebUI、AnythingLLM%20的建議.md) 新增「config key 與記憶 vs 知識庫邊界」；README 補上對話與記憶說明與新 key。

## 1.3.3（模型寫檔與寫入知識庫）

- **寫本機檔**：使用者輸入「寫 路徑」或「寫入 路徑」（支援 `~`、引號包住含空格路徑），系統在串流結束後將模型本則回覆的完整內容寫入該路徑；路徑**限工作區內**，否則略過並 log。可撰寫文檔、摘要等。
- **寫入知識庫**：使用者說「加入知識庫」「寫入知識庫」「存進知識庫」等，系統在串流結束後將模型本則回覆寫入知識庫（來源標為「對話寫入」）。模型可依附檔或對話產出摘要／要點供存入。
- **能力邊界**：能力邊界新增「寫本機檔」、將「寫入知識庫」改為可執行（系統代為寫入），模型不再聲稱無法寫入。

---

## 1.3.2（知識庫儲存位置可見）

- **知識庫路徑**：設定載入後將 `KnowledgePath` 轉為絕對路徑；啟動時即建立 `knowledge` 目錄；log 與 API 回傳儲存位置。
- **API**：`GET /api/knowledge/sources` 新增 `storage_path`（絕對路徑）；前端「📚 知識庫」面板顯示「儲存位置：…」。
- **Store**：`dir` 為空時 fallback 至工作目錄或執行檔目錄下的 `knowledge`，並轉絕對路徑；API 空路徑時回傳「(未設定)」；前端無路徑時顯示「(無法取得路徑)」。

---

## 1.3.1（網搜引用來源）

- **網搜結果附上引用連結**：後端搜尋 API（Brave / Tavily）改為回傳 `Result{Title, URL, Content}`；答覆串流結束前會多送一筆 SSE `sources`，前端在該則訊息下方顯示「引用來源」區塊，每筆為可點連結（`target="_blank"`），避免模型編造連結。
- **能力邊界**：若有即時搜尋結果，指示模型僅依該區塊作答，文末引用由系統自動附上、勿編造。

## 1.3.0（模型知識庫）

- **模型知識庫**：RAG 式攝入、檢索、注入；選配 Obsidian 風格 `[[連結]]` 解析與沿連結擴散檢索（蒲公英）。
- **儲存**：`{workspace}/knowledge/chunks.jsonl`，每條含 id、source、content、created_at、links。
- **攝入**：上傳檔案（附檔勾選「加入知識庫」）、貼文、URL；API `POST /api/knowledge/ingest`（form: file | text | url）。
- **檢索**：每輪答覆前依使用者訊息關鍵字檢索，top-k；可選沿 links 擴散。Config：`CHATMERY_KNOWLEDGE_*`（enabled、path、top_k、chunk_runes、overlap、expand_links、expand_max）。
- **前端**：附檔時可勾選「加入知識庫」；「📚 知識庫」按鈕可列出來源並刪除。設計詳見 [模型知識庫—設計提案](模型知識庫—設計提案.md)。

## 1.2.2（Markdown／富文字）

- **阿卅回覆**：以 Markdown 渲染（`**粗體**`、`*斜體*`、`` `程式碼` ``、換行），不再顯示裸的 `**`。
- 前端 `renderMarkdown()`、`.msg-body-md` 樣式；錯誤類訊息維持純文字。

---

## 1.2.1（輸出語言）

- **CHATMERY_OUTPUT_LANG**：回覆一律使用的語言（如「繁體中文」）；留空不限制，預設「繁體中文」。寫入能力邊界：「請一律使用「{值}」回覆。」
- Tuning / 環境變數皆可設；留空表示不限制。

---

## 1.2.0（Cursor 式網搜、版次模組）

- **搜尋 query 由 LLM 產出**：`CHATMERY_SEARCH_QUERY_LLM=true` 時，網搜前用同一模型依使用者句意產出單行搜尋 query（解讀意圖、偏好英文查詢、必要時加年份），減少無用關鍵字。
- **internal/search/prompt.go**：`SearchQuerySystemPrompt`、`QueryGenUserPrompt`；邏輯可透過修改 prompt 調教。
- **internal/version**：版次常數（如 `1.2.0`），用於 log 與 User-Agent；前端快取以 `?v=` 區分。

---

## 架構與設定對照

- **網搜**：Brave / Tavily，query 可選 LLM 產生；結果含 Title / URL / Content，引用由系統附上。
- **輸出**：可強制語言；回覆可 Markdown 渲染；每則顯示模型名。
- **知識庫**：獨立于記憶（四池一魂），可選啟用、chunk、擴散檢索；API 與 UI 支援攝入與來源管理。
