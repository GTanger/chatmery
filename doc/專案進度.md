# Chatmery 專案進度

> 供快速掌握現況與後續可做項目；可隨開發更新。

---

## 專案概述

- **定位**：對話外殼（Telegram + Ollama）+ 記憶系統（背版 + archival），Go 實作，預設為小模型（約 4B）設計。
- **優先級**：① 上網檢索 ② 寫文檔／紀錄討論。

---

## 已完成

| 項目 | 說明 |
|------|------|
| **Telegram Bot** | 指令解析、Ollama 串流回覆、/restart、開機自啟（systemd / run.sh） |
| **背版** | SOUL.md、MEMORY.md 載入與注入 |
| **長期記憶（archival）** | JSONL 儲存、關鍵字檢索、rollover（短期事實提煉後寫入） |
| **向量檢索** | 設 `CHATMERY_EMBED_MODEL` 啟用；archival_vectors.jsonl、Backfill、語意相似度 + 關鍵字 fallback |
| **網搜** | Brave / Tavily、關鍵字萃取、結果注入 |
| **讀檔** | 「讀／讀取 路徑」、路徑不鎖工作區、~ 與引號支援 |
| **讀檔格式** | PDF、.txt、.md、.docx、.xlsx、.odt、.ods（單檔約 28000 字上限） |
| **Telegram 附檔** | 同上格式，須 Caption，下載後擷取文字注入 |
| **寫檔** | 「寫／寫入 路徑 內容」、限工作區內、自動建目錄 |
| **設定** | .env、chatmery.tuning（可調模型、記憶條數、提煉門檻等） |

---

## 未完成／可擴充

| 項目 | 說明 |
|------|------|
| **讀圖** | 圖片（.jpg、.png 等）目前不支援；可選：OCR（如 Tesseract）或視覺模型（Ollama 多模態／外部 API） |
| **搜尋後端** | 已支援 Brave、Tavily（需 API key）。DuckDuckGo 未實作，可選擴充（通常需 scraper） |
| **記憶服務化** | internal/memory 可抽出為獨立 HTTP API，供遊戲或 OpenClaw 呼叫 |

---

## 模組一覽

| 路徑 | 職責 |
|------|------|
| `cmd/chatmery` | Telegram 入口、prompt 組裝、讀寫／附檔意圖、記憶與搜尋接線 |
| `internal/config` | 環境變數與路徑（WORKSPACE、TOKEN、搜尋、embed 等） |
| `internal/memory` | 背版、archival（JSONL + 向量）、session facts、rollover |
| `internal/search` | 關鍵字萃取、Brave / Tavily API |
| `internal/ollama` | Chat 串流、generate、embedding |
| `internal/document` | 讀檔擷取（PDF、txt、md、docx、xlsx、odt、ods） |
